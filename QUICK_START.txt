───────────────────────────────────────────────────────────────────
STEP 1: ONE-TIME SETUP (5 minutes)
───────────────────────────────────────────────────────────────────

npm install --legacy-peer-deps

cd backend
python -m venv venv
source venv/bin/activate          # macOS/Linux
  OR
venv\Scripts\activate             # Windows
pip install -r requirements.txt

───────────────────────────────────────────────────────────────────
STEP 2: RUN YOUR APP (every time you want to develop)
───────────────────────────────────────────────────────────────────

TERMINAL 1: Backend
════════════════════
cd backend
source venv/bin/activate          # macOS/Linux
npm start doesn't work here, skip this step on Windows

python -m uvicorn app.main:app --reload


TERMINAL 2: Frontend
════════════════════
(in a NEW terminal at project root)

npm start

Then choose:
  - Press 'i' for iOS Simulator
  - Press 'a' for Android Emulator  
  - Press 'w' for Web Browser
  - Scan QR with Expo app on phone

───────────────────────────────────────────────────────────────────
STEP 3: TEST THAT IT WORKS
───────────────────────────────────────────────────────────────────

In a third terminal, test the backend:

curl http://localhost:8000/health

Expected: {"status": "ok", ...}

───────────────────────────────────────────────────────────────────
STEP 4: USE THE APP
───────────────────────────────────────────────────────────────────

1. Open the app (iOS Simulator / Android Emulator / Web)
2. Press the microphone button to start recording
3. Sing or hum something
4. Press to stop recording
5. Watch it process:
   - AutoTune your pitch
   - Detect chords
   - Generate arrangements
6. Play the different versions (A-F)
7. Select your favorite to save

That's it! 🎵

───────────────────────────────────────────────────────────────────
DEPLOYING TO PRODUCTION
───────────────────────────────────────────────────────────────────

When you're ready to deploy:

npm install -g @railway/cli
railway login
cd backend
railway init
railway add       # Creates database
railway up        # Deploys your backend

Then update src/screens/StudioScreen.tsx with your production URL.

Full instructions: See COMMANDS.md

───────────────────────────────────────────────────────────────────
TROUBLESHOOTING
───────────────────────────────────────────────────────────────────

Backend won't start?
→ Check: python -m venv venv && source venv/bin/activate
→ Check: pip install -r requirements.txt
→ Check: python verify_system.py

Frontend won't compile?
→ Check: npm install --legacy-peer-deps
→ Check: Make sure no TypeScript errors
→ Check: npm start -- --reset-cache

Backend returns error?
→ Check: curl http://localhost:8000/health
→ Check: Python errors in terminal
→ Check: Environment variables (.env files)

Audio upload fails?
→ Check: File is .m4a or .mp3
→ Check: File size < 10MB
→ Check: Both backend and Supabase are running
→ Check: BACKEND constant in StudioScreen.tsx

WHAT YOUR APP DOES
───────────────────────────────────────────────────────────────────

✅ Records your voice
✅ Auto-tunes your pitch (PYIN algorithm)
✅ Detects what key you're singing in
✅ Finds the chords (HMM music theory)
✅ Generates 6 different backing arrangements:
   A = Ballad (slow, simple)
   B = Folk (acoustic guitar style)
   C = Full Band (drums + bass + leads)
   D = Cinematic (orchestral, slow)
   E = Lo-Fi (chill hip-hop vibes)
   F = Indian (tabla + sitar style)

✅ All audio saved to cloud (Supabase)
✅ Works on iOS, Android, and Web
✅ Completely offline capable (except cloud sync)
